# Engineering Baseline

## Purpose

Define the minimum engineering baseline for YearChart.

## Decisions

* Use GitHub Actions for CI/CD.
* Run build, test, secret scanning, and dependency update automation in GitHub Actions.
* Target the latest supported .NET LTS release.
* Treat YearChart as a Windows-only desktop application.
* Complete a CHANGELOG.md file ensuring each release is documented, with a summary of any earlier, previously undocumented releases.
